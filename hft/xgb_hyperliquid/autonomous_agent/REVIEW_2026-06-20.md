## Confirmed BLOCKERS (ranked by directness to losing money)

1. **Exit path books PnL + marks position `closed` BEFORE confirming the fill, never retries, never reconciles.**
   `xgb_bot.py:796` sets `pos.state="closed"` before the exit order at `:801`; `:816-817` book
   `cumulative_pnl`/`total_trades` outside the `filled` branch. `get_positions()` (`:273`) is
   never called. A failed/partial exit leaves a REAL naked position the bot thinks is closed.
2. **Partial-fill / non-reduce-only exit can FLIP into an opposite position.**
   Entry/exit use `reduce_only=False` (`:304`,`:801`); `_parse_result` (`:362-365`) never reads
   filled size, so a partial entry is recorded at full size and the full-size "close" over-closes.
3. **Kill-switch is an in-memory estimate that resets to 0 on every restart and is never persisted.**
   `:854` halts on `cumulative_pnl < -max_loss_usd`, computed from mids + constant 6.48, RAM-only.
   A crash/restart loop bypasses the cap entirely. (A soft halt DOES exist — "no kill-switch at
   all" was an overstatement — but it is not durable and not externally enforced.)
4. **Median fallback is dead code: ~17/79 v8 features reach the model as `0.0`, not their training medians.**
   `xgb_feature_engine.py:917` returns `result[f]=all_feats.get(f,0.0)` for every name, so
   `predict`'s `medians.get(f,…)` (`xgb_bot.py:128`) never fires. Empirically 17/79 came back 0.0;
   14 have non-trivial medians (`vol_proxy_1m`≈2990, `bid_depth_s`≈635). OOD inputs on every model.
   Mitigant: all are low XGBoost importance (~1-3% gain) — bad, not necessarily catastrophic.
5. **Watchdog respawns the WRONG model set → infinite restart→exit(1) loop (or trades a stale portfolio).**
   `watchdog.sh:66` hardcodes `--models_dir models/live_v3`, but `main()` requires
   `sol/short_1m_tp0`+`short_1m_tp2`, which only `live_v8` has. After any crash the bot can never
   restart cleanly.
6. **Credential path mismatch + isolation hazard (design-critical).**
   `xgb_bot.py:494-496` reads `/bot/hl/*`, but the experiment IAM role allows only `/agent/hl/*`
   and HARD-DENIES `/bot/*` (`iam.tf:41-51`). So the bot cannot start on the capped box. The
   dangerous "fix" would be widening IAM, which would expose the MAIN $403 wallet key to the
   capped box and defeat the entire wall-off. Correct fix: parameterize the bot to read `/agent/hl/*`.

**Also money-relevant (MAJOR):** no leverage/margin mode is ever set (`xgb_bot.py` has zero
`update_leverage`/`isolated` calls) — a $50 cross-margin wallet can be fully liquidated on one
SOL gap. You cannot lose MORE than $50 (no clawback), but the whole $50 can vanish fast.

## Remediation round (2026-06-20, same day) — blockers 1-4 + 6 fixed in code

Edited (working tree, NOT committed): `xgb_bot.py`, `xgb_feature_engine.py`.

| Blocker | Fix |
|---|---|
| 1 (exit abandons positions) | Close placed first; PnL/counters/`closed` only on confirmed fill; failed exit stays OPEN, retries, escalates to `market_close` once, halts after 5 attempts; orphan positions reconciled at startup. |
